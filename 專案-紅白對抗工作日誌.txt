### [2026-03-13 修復漏洞紀錄]

#### 🛠️ 今日修復目標：多人併發投票異常處理
根據上週實戰測試回饋，當多人同時湧入投票時，系統會出現以下嚴重問題：
1. **網頁自動刷新**：投票動作觸發了不必要的頁面重導或刷新。
2. **票數歸零**：高併發下，伺服器端的變數處理可能產生競爭條件 (Race Condition)，導致數據遺失。
3. **匯入檔案狀態消失**：系統狀態管理不穩定，導致本應存在的匯入數據在併發過程中遺失。

#### 🔍 預計處理方向：
- 檢查 `main.py` 中的全局變數使用情況，評估是否需要鎖定機制或改用資料庫存儲。
- 優化 frontend `index.html` 的 AJAX 請求，防止表單預設提交導致的刷新。
- 檢查 Vercel 無伺服器環境 (Serverless) 的狀態持久化限制。

#### ✅ 今日已完成修復：
1. **實作 Append-only 投票機制**：
   - 新增 `VotesLog` 分頁。投票時改為 `append_row` 紀錄，徹底杜絕多人在同一儲存格競爭導致票數歸零。
   - 後端 `/status` 與 `/end_round` 會自動彙整該回合的所有得票數。
2. **強化 Serverless 狀態持久化**：
   - 在所有關鍵進入點（`/status`, `/admin`, `/vote` 等）加入 `load_data()` 強制檢查。
   - 當 Vercel 實例重啟導致記憶體清空時，系統能立即從 Google Sheets 恢復作品清單，解決「匯入狀態消失」問題。
3. **前端穩定性優化**：
   - 加入投票按鈕鎖定 (isVoting 旗標)，防止重複點擊。
   - 加入投票中 Loading 提示。
   - 優化狀態輪詢 (checkStatus) 的錯誤容忍度，避免網路波動導致 UI 異常。
4. **版本更新至 v1.1.9**：
   - 更新 `UPDATE_LOG` 與 `APP_VERSION` 以識別修復版本。

### [2026-03-18 系統穩定性優化與同步機制修復紀錄]

#### 🛠️ 今日目標：解決 Google API 限流、重設不同步與即時票數顯示問題
1. **解決 429 錯誤**：在高併發狀態輪詢下，Google Sheets API 出現配額限制。
2. **重設同步機制**：解決「老師按重設，學生手機端沒反應」的狀態不同步問題。
3. **即時數據更新**：讓老師主控台能即時看到投票數據，而不受緩存影響。

#### ✅ 今日已完成修復：
1. **實作記憶體快取 (In-memory Cache)**：
   - 在 `main.py` 實作了 12-15 秒的同步間隔。伺服器不再每次輪詢都讀取 GSheet，而是從記憶體回傳，大幅降低 API 調用頻率。
2. **System ID 雲端持久化**：
   - 將 `system_id` 存入 Google Sheets。解決了 Vercel 多實例環境下 SID 不一致的問題。
   - **重設偵測**：當老師在本地端執行「重設系統」，會更新雲端 SID。學生端（Vercel）讀取到新 ID 後，會強迫手機瀏覽器清除 `localStorage` 並重新載入，實現「全網重置」。
3. **老師專屬即時通道 (`/admin_status`)**：
   - 替主控台建立了獨立的即時數據路由，繞過所有緩存直接讀取 GSheet。這讓老師能每 3 秒看到最準確的得票跳動，同時學生端繼續使用緩存保護 API。
4. **Vercel 環境適應性修正**：
   - 修正了 `python-dotenv` 在 Vercel 環境缺失導致的啟動失敗 (v1.2.1)。
   - 更新 `requirements.txt` 並優化 `main.py` 的導入邏輯。
5. **版本更新至 v1.2.2**：
   - 全面打通「本地管理 - 雲端投票」的雙向同步。

### [2026-03-18 晚間 修復同步與重設漏洞]

#### 🛠️ 今日修復目標：解決手機端卡死與重設無效問題
1. **修復 429 卡死迴圈**：修正 `/status` 在遇到 GSheet API 限流時未更新快取時間，導致 DDOS 迴圈的 Bug。
2. **修正重設邏輯**：修正 Vercel 環境下錯誤從 `/tmp` 還原舊資料至 GSheet 的行為。
3. **優化併發效能**：將所有阻塞型路由移至 Threadpool 執行，解決非同步阻塞問題。

#### ✅ 今日已完成修復：
1. **強化快取防護機制**：在 `main.py` 的同步邏輯中加入 `finally` 區塊，確保無論成功或失敗皆會進入 12 秒冷卻期，成功解除 API 限流死鎖。
2. **禁用 Vercel 本地還原**：修改 `load_data()`，在 Vercel 環境下嚴禁從本地 JSON 備份還原資料，確保「重設系統」後資料不會被意外救回。
3. **移除路由 `async` 宣告**：針對所有涉及 GSheet I/O 的路由移除 `async` 關鍵字，交由 FastAPI 執行緒池處理，大幅提升多人併發時的反應速度。
4. **優化 GSheet API 使用效率**：合併「系統狀態」與「SystemID」的讀取請求為單一 API 呼叫，減少 1/3 的 API 消耗，顯著降低 429 錯誤機率。
5. **調整主控台更新頻率**：將老師主控台的更新頻率從 3 秒延長至 5 秒，確保在高頻投票時仍能守住 Google API 的 60次/分 配額限制。
6. **強化 `load_data()` 防護與重試機制**：加入 10 秒載入冷卻時間，並修正載入失敗時仍標記為「已載入」的 Bug。現在若 API 故障，系統會維持現狀而非誤報「空資料」，徹底解決「系統自動歸零」的問題。
7. **版本更新至 v1.2.3**。
